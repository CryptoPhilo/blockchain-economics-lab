# CRO-001: Gemini Deep Research v2 파이프라인 도입 검토

## Executive Summary

BCE-353에서 제출된 Gemini Deep Research v2 프롬프트를 평가하기 위해 동일 프로젝트(코스모스)에 대한 ECON 보고서를 기존 파이프라인과 v2로 각각 생성하여 비교 분석을 수행했습니다.

**핵심 권고**: ❌ **현 시점에서 v2 프롬프트로 프로덕션 파이프라인 교체 불가**

**주요 근거**:
- v2 보고서는 **학술적 인용 표준 미충족** (참고문헌 섹션 없음, URL 0개)
- 내용의 깊이는 우수하나 **연구 무결성 검증 불가능**
- BCE Lab의 글로벌 학술 신뢰도 유지를 위해 현 파이프라인 유지 필요

---

## 1. 배경

### 1.1 분석 목적
BCE-353에서 제출된 "Gemini Deep Research v2" 프롬프트가 기존 Anthropic Claude 기반 딥리서치 파이프라인을 대체할 수 있는지 검증

### 1.2 분석 방법
- **대상 프로젝트**: 코스모스 (Cosmos) 블록체인
- **보고서 유형**: ECON (Economic Analysis Report)
- **비교 대상**:
  - 원본: 기존 파이프라인으로 생성한 코스모스 ECON (11페이지, 48개 출처)
  - V2: Gemini Deep Research v2 프롬프트로 생성한 코스모스 ECON (20페이지)

### 1.3 분석 기준
1. 기술적 깊이 및 정확성
2. 연구 엄격성 (학술적 인용 표준)
3. 출처 검증 가능성
4. 구조 및 가독성
5. 실용성 (QA 가능성, 감사 가능성)

## 3. 정성 평가

### 3.1 V2의 주요 강점

#### 1) 온체인 상태 매핑의 정확성
- 예시: "x/staking 모듈의 Validator 구조체에 기록되며, 0x21 접두사를 가진 키로 인덱싱"
- 원본: "x/staking 모듈에 의해 온체인 상에서 엄격하게 통제된다"
- → **V2가 프로토콜 레벨 정확도 높음**

#### 2) 체계적 개념 정의
- 각 개념마다 4단계 구조:
  1. 온체인 매핑 (어디 저장되는가)
  2. 정의 (무엇인가)
  3. 기능 (무엇을 하는가)
  4. 출처 (근거는 무엇인가)
- → **교육 자료로서 가치 높음**

#### 3) 비교 분석 및 평가
- Cosmos vs Polkadot vs Ethereum 비교 테이블
- 5개 차원 별점 평가 (가치/보상/토큰/부트스트랩/지속가능성)
- → **투자자 의사결정에 유용**

### 3.2 V2의 치명적 약점

#### 1) ❌ 학술적 인용 부재
**문제점**:
- 48개 URL 기반 참고문헌 → 0개
- 검증 가능한 출처 → 검증 불가능한 일반 참조
- 접근 날짜 기록 → 전혀 없음

**영향**:
- 외부 연구자가 출처 확인 불가능
- 동료 검토(peer review) 불가능
- 학술 논문/보고서 인용 불가능
- 법적 분쟁 시 근거 제시 불가

#### 2) ❌ 할루시네이션 검증 불가
**위험**:
- Gemini가 학습 데이터에서 합성한 정보일 가능성
- 실제 문서가 없는 "출처: 코스모스 백서 Section 4" 참조 가능성
- URL 없이는 사실 확인 불가능

**사례** (가정):
- V2: "Gini 수수료/발행 비율 0.1" (출처 없음)
- → 이 수치가 정확한지 독자가 확인할 방법 없음

#### 3) ❌ QA 프로세스 적용 불가
**현 BCE Lab QA 프로세스**:
1. 보고서 수령
2. 참고문헌 URL 무작위 샘플링 (10~20%)
3. 원본 출처와 보고서 주장 대조
4. 오류 발견 시 수정 요청

**V2의 경우**:
- URL 0개 → QA 단계 2~3 실행 불가능
- 품질 보증 불가능

## 4. 근본 원인 분석

### 4.1 왜 Gemini v2는 인용을 포함하지 않는가?

**가설 1: 설계상 한계**
- Gemini Deep Research는 "합성(synthesis)" 최적화
- 정보 출처 추적(source tracking)은 부차적 목표
- 학습 데이터에서 패턴 추출 → URL 연결 유지 안 됨

**가설 2: 실시간 웹 액세스 제한**
- 정적 학습 데이터에 의존
- 최신 문서 URL 실시간 검증 불가
- 환각(hallucination) 위험 회피 위해 URL 생략 선택

**가설 3: 프롬프트 누락**
- v2 프롬프트에 "참고문헌 포함" 명시 누락 가능성
- 개선 가능성 있음 (후속 테스트 필요)

### 4.2 원본 파이프라인의 인용 메커니즘

**Anthropic Claude 기반 파이프라인**:
1. 웹 검색 수행 (Perplexity/Tavily 등)
2. 관련 문서 수집 및 URL 저장
3. 콘텐츠 생성 시 출처 번호 매핑
4. 참고문헌 섹션 자동 생성

→ **출처 추적이 파이프라인에 내장**

## 6. CRO 권고사항

### 6.1 즉시 조치: ❌ V2 도입 보류

**근거**:
1. BCE Lab의 핵심 자산은 "신뢰할 수 있는 글로벌 크립토 리서치"
2. 인용 없는 보고서는 학술/투자 커뮤니티에서 신뢰 상실
3. 리스크가 이익을 압도적으로 초과

### 6.2 중기 대안: 하이브리드 접근

**Option A: 구조 차용 + 인용 수동 추가**
1. V2의 우수한 구조(온체인 매핑 4단계)를 템플릿으로 채택
2. 콘텐츠는 기존 Claude 파이프라인으로 생성
3. 48+ URL 인용 유지

**예상 효과**:
- 기술적 깊이: ⭐⭐⭐⭐ → ⭐⭐⭐⭐½
- 연구 엄격성: ⭐⭐⭐⭐⭐ (유지)
- **Best of both worlds**

**추가 작업량**: 템플릿 개발 2일 + 테스트 3건

### 6.3 장기 대안: 2단계 파이프라인

**설계**:
```
Stage 1 (Gemini v2):
  입력: 프로젝트 기본 정보
  출력: 상세 기술 분석 (20페이지, 인용 없음)
  
Stage 2 (Claude + Web Search):
  입력: Stage 1 출력
  작업: 
    - 각 주장에 대해 웹 검색 수행
    - 출처 URL 수집 및 검증
    - 참고문헌 섹션 생성
  출력: 완성 보고서 (20페이지 + 48+ URL)
```

**장점**:
- Gemini의 기술적 깊이 활용
- Claude의 출처 추적 능력 활용
- 품질 보증 유지

**단점**:
- 파이프라인 복잡도 2배
- 비용 증가 (2개 LLM 사용)
- 개발 시간 2주 예상

**ROI 분석**:
- 보고서 품질 향상: 15~20%
- 비용 증가: 50~70%
- **현 시점에서 ROI 불명확** → 추가 분석 필요

### 6.4 단기 실험: 프롬프트 엔지니어링

**가설**: v2 프롬프트에 명시적 인용 요구 추가로 개선 가능

**실험 계획**:
1. v2 프롬프트에 추가:
   ```
   "CRITICAL REQUIREMENT:
   - Include full bibliography section with URLs
   - Cite minimum 40 sources with accessible links
   - Format: [Number]. Title - Publisher, Access Date, URL"
   ```
2. 테스트 프로젝트: Polygon (기존 ECON 있음)
3. 비교: 개선된 v2 vs 기존 Polygon ECON
4. 판단 기준: URL 개수 >= 40

**타임라인**: 1주
**책임자**: CRO
**Decision gate**: 테스트 통과 시 Option C (2단계 파이프라인) 재평가
