# CMO Board Report MKT-005
## 소형 신규 프로젝트 커버리지 전략 검토

> **Report ID**: MKT-005 | **Date**: 2026-04-12 | **Author**: CMO Division
> **Classification**: INTERNAL — CONFIDENTIAL

---

## 1. Executive Summary

CEO의 지시에 따라, BTC·ETH·Uniswap 등 대형 프로젝트 대신 **소형 신규 프로젝트에 집중하는 커버리지 전략**의 마케팅적 유효성, 지속 가능성, 실행 리스크를 분석합니다.

### 결론 요약

| 관점 | 평가 |
|------|------|
| 신규 사용자 유입 효과 | **높음** — 경쟁 공백 영역, "알파 헌팅" 수요 강력 |
| 지속 가능성 | **제한적** — 프로젝트 생존율 ~50%, 검색 볼륨 변동 극심 |
| 경쟁 차별화 | **매우 높음** — Delphi/Messari/Nansen 모두 대형 위주, 커버리지 공백 |
| 데이터 수집 가능성 | **도전적** — 소형 프로젝트는 API 데이터, 온체인 히스토리 부족 |
| 브랜드 리스크 | **중간** — 분석 대상이 실패/스캠 시 신뢰도 리스크 |

**CMO 권고**: 소형 신규 프로젝트 전용 전략이 아닌, **"Discovery + Established" 하이브리드 모델** 채택을 권고합니다. 대형 프로젝트로 신뢰 기반을 확보하고, 소형 신규 프로젝트로 차별화된 유입을 견인하는 구조입니다.

### 2.2 수요 측 분석

소형/신규 프로젝트에 대한 리서치 수요는 두 가지 사용자 유형에서 발생합니다.

**유형 A: "알파 헌터(Alpha Hunter)"**
- 프로필: 경험 있는 크립토 트레이더, 대형 프로젝트 대비 초과 수익 추구
- 수요: TGE 직후 프로젝트의 토크노믹스 검증, 언락 리스크, 초기 고래 움직임
- 지불 의향: 높음 — "남들이 모르는 분석"에 프리미엄 지불 가능
- 규모: 크립토 액티브 트레이더의 약 15-20% (추정)

**유형 B: "리스크 회피 초기 투자자"**
- 프로필: 소형 프로젝트에 관심 있으나 스캠/러그풀 우려
- 수요: "이 프로젝트가 안전한가?" — 포렌식 + 성숙도 평가
- 지불 의향: 중간 — 무료 콘텐츠로 유입, 리스크 경보에 유료 전환
- 규모: 신규 프로젝트 투자자의 약 30-40% (추정)

**검색 키워드 볼륨 비교 (추정치):**

| 키워드 패턴 | 월간 검색량 | 경쟁도 | 전환 의도 |
|------------|:---:|:---:|:---:|
| "[대형 프로젝트] tokenomics analysis" | 2K-10K | HIGH | MEDIUM |
| "[신규 프로젝트] review" | 5K-50K (출시 직후 스파이크) | LOW | HIGH |
| "[신규 프로젝트] scam or legit" | 10K-100K (스파이크) | LOW | VERY HIGH |
| "new crypto projects to invest 2026" | 20K-50K | HIGH | MEDIUM |
| "low cap crypto analysis" | 3K-8K | MEDIUM | HIGH |

**핵심 인사이트**: 신규 프로젝트 출시 직후 "[프로젝트명] review", "[프로젝트명] scam or legit" 검색이 폭발적으로 증가합니다. 이 시점에 BCE Lab 보고서가 존재하면 오가닉 유입 효과가 대형 프로젝트 분석 대비 **5-10배** 클 수 있습니다. 다만 이 검색량은 **일시적 스파이크**이며 2-4주 후 급감합니다.

### 2.3 리스크 측 분석

| 리스크 | 심각도 | 설명 |
|--------|:---:|------|
| **프로젝트 실패/스캠** | HIGH | 신규 프로젝트의 약 50%가 1년 내 실패하거나 스캠으로 판명. BCE Lab이 분석한 프로젝트가 스캠이면 신뢰도 타격 |
| **데이터 부족** | HIGH | 온체인 히스토리 짧음, API 미제공, GitHub 비공개, 토큰 분배 불투명 — BCE Maturity Score™ 7축 산출에 근본적 한계 |
| **검색 볼륨 변동성** | MEDIUM | 대형 프로젝트는 안정적 검색 볼륨, 소형은 출시 직후만 스파이크 → 롱테일 SEO 효과 약함 |
| **법률 리스크** | MEDIUM | 미등록 증권(Howey 테스트) 가능성 있는 프로젝트 분석 시 규제 리스크 |
| **확장 비용** | MEDIUM | 신규 프로젝트 속도에 맞추려면 리서치 파이프라인의 턴어라운드 시간을 대폭 단축해야 함 |

## 3. 하이브리드 모델 제안: "Discovery + Established"

### 3.1 모델 개요

소형 프로젝트만 커버하는 것은 지속 가능성 리스크가 크므로, **대형 앵커 프로젝트 + 소형 발굴 프로젝트를 전략적으로 배합**하는 하이브리드 모델을 제안합니다.

```
┌─────────────────────────────────────────────────────────┐
│            BCE Lab Coverage Portfolio                    │
│                                                         │
│  ┌──────────────────┐   ┌───────────────────────────┐   │
│  │  ESTABLISHED     │   │  DISCOVERY                │   │
│  │  (앵커 프로젝트)   │   │  (발굴 프로젝트)           │   │
│  │                  │   │                           │   │
│  │  시총 Top 100    │   │  시총 100위 밖            │   │
│  │  TGE 6개월+      │   │  TGE 6개월 미만           │   │
│  │                  │   │  온체인 홀더 급증 중       │   │
│  │  역할:           │   │  거래량 상위 신규 토큰     │   │
│  │  · 신뢰 기반     │   │                           │   │
│  │  · 안정적 SEO    │   │  역할:                    │   │
│  │  · 구독 유지     │   │  · 차별화 유입            │   │
│  │  · BCE Score     │   │  · 바이럴 촉발            │   │
│  │    기준점        │   │  · 알파 헌터 포착          │   │
│  │                  │   │  · 경쟁 공백 장악          │   │
│  │  비중: 60%       │   │  비중: 40%                │   │
│  └──────────────────┘   └───────────────────────────┘   │
│                                                         │
│  배합 비율: Phase별 조정 가능                             │
│  Phase 1: 70:30 (신뢰 구축 우선)                         │
│  Phase 2: 60:40 (발굴 확대)                              │
│  Phase 3: 50:50 (균형)                                   │
└─────────────────────────────────────────────────────────┘
```

### 3.2 Discovery 프로젝트 선정 기준

모든 소형/신규 프로젝트를 커버하는 것은 리소스 낭비이므로, 다음 기준으로 선별합니다.

**필수 조건 (4개 모두 충족):**

| # | 기준 | 측정 방법 | 근거 |
|---|------|----------|------|
| 1 | 24h 거래량 $5M 이상 | CoinGecko API | 최소 유동성 + 사용자 관심 증거 |
| 2 | 온체인 홀더 수 1,000 이상 또는 급증 추세 | Etherscan/Explorer API | 실사용 증거, 봇이 아닌 실제 채택 |
| 3 | 스마트 컨트랙트 검증됨 (Verified on Explorer) | Block explorer | 최소 투명성 기준 |
| 4 | 팀 또는 DAO 정보 공개 | 수동 확인 | 익명 러그풀 리스크 배제 |

**가산 점수 (2개 이상 충족 시 우선순위 상향):**

| # | 기준 | 가산 | 근거 |
|---|------|:---:|------|
| A | GitHub 퍼블릭 리포 + 월 50+ 커밋 | +2 | 기술 활동 증거 → MAT 스코어링 가능 |
| B | 유명 VC 시드/시리즈 투자 | +2 | 최소 듀딜 통과 증거 |
| C | 멀티체인 배포 또는 L2 론칭 | +1 | 기술적 성숙도 |
| D | 거래소 2개 이상 상장 | +1 | 유동성 + 접근성 |
| E | 소셜 센티먼트 급상승 (X/TG) | +1 | 마케팅 적시성 |

**배제 기준 (1개라도 해당 시 커버하지 않음):**

| # | 기준 | 근거 |
|---|------|------|
| X1 | 밈코인 순수 투기 | BCE Lab 브랜드 보호 — "리서치"와 불일치 |
| X2 | KYC/팀 정보 완전 불명 | 러그풀 리스크 너무 높음 |
| X3 | 토큰 분배 90%+ 인사이더 | 투자자 보호 관점 불가 |
| X4 | 법적 분쟁 진행 중 (SEC 등) | 법률 리스크 |

### 3.3 Established vs Discovery 콘텐츠 차이

| 차원 | Established (대형) | Discovery (소형/신규) |
|------|:---:|:---:|
| 보고서 깊이 | Full (ECON v2 11챕터) | **Compact** (6-7챕터, 핵심만) |
| BCE Maturity Score™ | 7축 전체 | **5축** (Compliance, Governance 데이터 부족 시 N/A 표기) |
| 업데이트 주기 | 3-6개월 | **1-2개월** (빠른 변화 반영) |
| 가격 | $49/$39/$29 | **$29 균일가** 또는 뉴스레터 프리뷰 |
| 무료 콘텐츠 비율 | 35-50% | **60-70%** (유입 최적화) |
| 제작 시간 | 5-7일 | **2-3일** (Compact 포맷) |
| SEO 수명 | 장기 (6개월+) | **단기** (2-4주 스파이크) |

### 3.4 "BCE Discovery Report" 신규 콘텐츠 유형

기존 ECON/MAT/FOR 3종과 별도로, 소형 신규 프로젝트 전용 **통합 요약 보고서 유형**을 신설합니다.

**BCE Discovery Report 구조:**

```
BCE Discovery Report: [Project Name]
TGE: [날짜] | Market Cap: [시총] | BCE Discovery Score: [점수]/100

Ch.1  TL;DR — 2분 요약 (무료)
Ch.2  프로젝트 개요 + 팀 배경 (무료)
Ch.3  토크노믹스 Quick Check (무료)
      · 공급 구조 / 언락 일정 / 인사이더 비율
Ch.4  초기 온체인 신호 (무료)
      · 홀더 분포 / 고래 집중도 / DEX vs CEX 유동성
─── Freemium 경계선 (60%) ───
Ch.5  경제 설계 리스크 평가 (ECON 축약)
Ch.6  초기 성숙도 평가 (MAT 축약, 5축)
Ch.7  포렌식 스캔 — 레드 플래그 체크리스트
      · 컨트랙트 소유권 / 민팅 권한 / 유동성 잠금 여부
      · 인사이더 거래 패턴 / 소셜 봇 활동
Ch.8  BCE Discovery Score + 비교 (동종 신규 3-5개)
Ch.9  Action Framework (관망/소량진입/회피 권고)

Appendix: Raw Data + Methodology + Disclaimer
```

**가격**: $29/건 또는 Project Sub($19/월)에 포함

**주요 차별화**: 기존 경쟁사는 이런 통합 평가를 신규 소형 프로젝트에 제공하지 않습니다. 특히 Ch.7 "포렌식 스캔 — 레드 플래그 체크리스트"는 신규 프로젝트 투자자가 가장 필요로 하면서 가장 부족한 정보입니다.

## 4. 데이터 수집 파이프라인 보강 방안

CEO 지시대로, 소형 신규 프로젝트는 데이터 수집이 어렵습니다. 이를 해결하기 위한 파이프라인 보강 방안입니다.

### 4.1 데이터 가용성 비교

| 데이터 항목 | 대형 프로젝트 | 소형 신규 프로젝트 | 갭 해소 방안 |
|------------|:---:|:---:|------|
| CoinGecko 가격/시총 | ✅ | ✅ (상장 시) | 그대로 사용 |
| Token Terminal 매출/TVL | ✅ | ❌ 대부분 미지원 | DefiLlama 직접 수집 |
| Tokenomist 언락 일정 | ✅ | ❌ 미등록 | 프로젝트 Docs에서 수동 파싱 |
| GitHub 활동 | ✅ | ⚠️ 비공개 리포 다수 | 공개 리포만 사용, 없으면 N/A |
| Nansen 지갑 레이블 | ✅ | ❌ 레이블 없음 | Etherscan/Explorer 직접 분석 |
| Dune 커스텀 쿼리 | ✅ | ⚠️ 데이터 제한적 | 온체인 직접 쿼리 |
| 소셜 센티먼트 | ✅ | ⚠️ 노이즈 多 | LunarCrush / 자체 필터링 |

### 4.2 신규 데이터 수집 레이어 설계

기존 파이프라인(RES-002 §6.2)에 다음 레이어를 추가합니다.

```
[기존 파이프라인 — Established 프로젝트용]
data-engineer/
  ├── price_daily         (CoinGecko)
  ├── onchain_daily       (Dune/Nansen)
  ├── whale_transfers     (Etherscan)
  └── market_snapshots    (CoinGecko)

[신규 레이어 — Discovery 프로젝트용]          ← NEW
discovery-pipeline/
  ├── new_token_scanner                       ← NEW
  │   · CoinGecko /coins/list → 신규 상장 토큰 일일 스캔
  │   · 필터: 24h volume > $5M, holders > 1K
  │   · 출력: discovery_candidates 테이블
  │
  ├── contract_analyzer                       ← NEW
  │   · Block Explorer API → 컨트랙트 소유권 확인
  │   · 민팅 권한, Proxy 여부, 유동성 잠금 확인
  │   · Honeypot/Rugpull 패턴 탐지
  │   · 출력: contract_risk_flags 테이블
  │
  ├── holder_distribution                     ← NEW
```

### 4.4 BCE Discovery Score 산출 방법 (데이터 부족 대응)

기존 BCE Maturity Score™는 7축 가중 평균이지만, 소형 신규 프로젝트는 Governance(10%)와 Compliance(10%) 데이터가 대부분 부재합니다. 이를 위한 적응형 스코어링을 설계합니다.

**방법: 가용 축 동적 가중치 재분배**

```
[일반 BCE Maturity Score™]           [BCE Discovery Score — 적응형]

Technology:  20%                     Technology:  25% (+5%)
Business:    20%                     Business:    20% (유지)
Tokenomics:  15%                     Tokenomics:  25% (+10%)  ← 신규에서 가장 중요
Governance:  10%                     Governance:  N/A (데이터 부족 시 제외)
Community:   10%                     Community:   10% (유지)
Compliance:  10%                     Compliance:  N/A (데이터 부족 시 제외)
Narrative:   15%                     Initial Traction: 20% (NEW — 대체 축)

Total: 100%                          Total: 100% (가용 축으로 재정규화)
```

**Initial Traction 축** (Narrative Health 대체):
- 첫 30일 홀더 증가율
- DEX 유동성 대비 거래량 비율
- 소셜 멘션 대비 실제 온체인 활동 비율
- 초기 고래 집중도 변화 추세

이렇게 하면 데이터가 부족해도 **의미 있는 정량적 평가**가 가능하며, "N/A" 축이 있음을 투명하게 표기하여 신뢰도를 유지합니다.

## 5. 마케팅 전략: Discovery 콘텐츠의 유입 효과 극대화

### 5.1 "First Mover" 콘텐츠 전략

소형 신규 프로젝트 분석의 핵심 가치는 **"최초 분석"**입니다. "[프로젝트명] 분석"을 검색할 때 BCE Lab이 첫 번째 결과로 노출되면 오가닉 유입이 폭발적입니다.

**실행 전략:**

| **실시간 레드 플래그 트윗** | 신규 프로젝트 컨트랙트 분석 결과 X(Twitter) 속보 | TGE 당일 | 바이럴 + 뉴스레터 CTA |

## 7. 리스크 완화 전략

### 7.1 브랜드 리스크 보호

소형 프로젝트 커버리지의 가장 큰 리스크는 **분석한 프로젝트가 실패/스캠**이 되었을 때의 신뢰도 타격입니다. 이를 완화하기 위한 방어 장치:

| # | 방어 장치 | 설명 |
|---|----------|------|
| 1 | **"분석 ≠ 추천" 면책** | 모든 Discovery Report에 "이 보고서는 투자 추천이 아닌 분석 자료이며, BCE Lab은 프로젝트의 성공을 보증하지 않습니다" 명시 |
| 2 | **레드 플래그 정직 공개** | 리스크를 발견하면 숨기지 않고 명확히 표기. "이 프로젝트에는 X 리스크가 있습니다" — 사후에 문제 발생 시 "우리는 경고했다"가 됨 |
| 3 | **Discovery Score 공개 방법론** | 점수 산출 근거를 투명하게 공개하여, 점수가 낮은 프로젝트도 커버하되 "왜 낮은지"를 설명 |
| 4 | **사후 추적 리포트** | 분석 3개월 후 "What Happened?" 팔로업 발행 → 예측 정확도 트래킹 공개 |
| 5 | **배제 기준 엄격 적용** | §3.2의 배제 기준을 예외 없이 적용. 밈코인/익명 팀/극단적 인사이더 비율은 절대 커버하지 않음 |

### 7.2 데이터 품질 리스크 보호

| 리스크 | 대응 |
|--------|------|
| 온체인 데이터 부족으로 BCE Score™ 산출 불가 | 적응형 스코어링(§4.4) 사용 + N/A 축 투명 표기 |
| 프로젝트 Docs 내용과 실제 불일치 | "Docs vs On-chain" 검증 섹션 필수 포함 |
| 소셜 봇 활동으로 인한 왜곡 | 봇 탐지 필터 적용, "Adjusted Social Score" 표기 |
| API 데이터 갱신 지연 | 최종 수동 확인 단계 추가 (CRO QA) |

소형 신규 프로젝트 커버리지는 BCE Lab에게 **명확한 경쟁 공백을 활용한 사용자 유입 전략**입니다. Delphi, Messari, Nansen 모두 Top 50 위주의 커버리지를 제공하며, 신규 소형 프로젝트에 대한 토크노믹스+성숙도+포렌식 통합 분석은 시장에 존재하지 않습니다.

그러나 CEO 지적대로 지속 가능성에 한계가 있고, 데이터 수집 파이프라인 보강이 필수적입니다. CMO는 다음 세 가지를 핵심으로 권고합니다.

첫째, **하이브리드 모델(Established 60% + Discovery 40%)을 채택**합니다. 대형 앵커 프로젝트로 BCE Lab의 분석 역량과 신뢰를 증명하고, Discovery 프로젝트로 경쟁사가 다루지 않는 영역에서 차별화된 유입을 견인합니다.

둘째, **"BCE Discovery Report"라는 신규 Compact 보고서 유형을 신설**합니다. 기존 Full Report 대비 60% 무료 비율, 2-3일 제작, $29 균일가로 빠른 턴어라운드와 접근성을 확보합니다. 특히 "포렌식 스캔 — 레드 플래그 체크리스트"는 신규 프로젝트 투자자가 가장 필요로 하는 콘텐츠입니다.

셋째, **데이터 수집 파이프라인을 단계적으로 보강**합니다. Phase 1에서 new_token_scanner와 contract_analyzer를 우선 구축하고($0 추가 비용), Phase 2-3에서 holder_distribution, social_signal_tracker, docs_parser를 순차 추가합니다(총 ~$280-380/월). BCE Discovery Score는 적응형 가중치로 데이터 부족 축을 투명하게 처리합니다.

이 전략이 실행되면 MKT-003의 뉴스레터 구독자 2,000명 목표를 2,500명으로 상향할 수 있으며, Q4 MRR $3,000 달성 확률도 높아집니다.
