# CMO Board Report MKT-006
## 전 종목 유니버설 커버리지 — 화제성 중심 시장 진입 전략

> **Report ID**: MKT-006 | **Date**: 2026-04-12 | **Author**: CMO Division
> **Classification**: INTERNAL — CONFIDENTIAL

## 1. Executive Summary

CEO의 전략 지시를 반영합니다. MKT-005의 "하이브리드 선별 커버리지"에서 **근본적으로 다른 접근**입니다.

**CEO 핵심 지시:**

> 모든 소형 프로젝트를 다 분석 시도하고 목록도 제시하면서, '이 프로젝트는 투명성이 낮아 분석이 불가능함' 같은 표식을 붙이면 프로젝트 관계자의 관심을 더 끌 수 있다. 시장 진입 초기에 유료 판매를 늘이는 것이 아니라 시장 내 화제성이 높은 서비스가 되는 마케팅 전략을 검토하라. 사실상 전세계 중앙화 거래소에서 거래되는 모든 종목에 대한 분석을 하는 유일한 크립토 리서치 서비스로 포지셔닝하라.

### 전략 전환 요약

| 항목 | MKT-005 (하이브리드) | **MKT-006 (유니버설 커버리지)** |
|------|:---:|:---:|
| 커버리지 범위 | 선별 (월 5-6건) | **CEX 상장 전 종목 (~2,000+ 토큰)** |
| 핵심 목표 | 유료 판매 + 유입 | **시장 화제성 + 인지도 → 추후 수익화** |
| 분석 방법 | 수동 Deep Analysis | **자동 스캔 + 등급 라벨링 + 선별 Deep** |
| 포지셔닝 | "360° Project Intelligence" | **"크립토의 무디스(Moody's)" — 전 종목 등급 부여** |
| 프로젝트 관계자 대응 | 없음 | **"투명성 낮음" 라벨 → 프로젝트 측 자발적 접근** |
| 수익 모델 | C2C (보고서 판매) | **C2C + B2B (프로젝트 등급 개선 컨설팅)** |

**CMO 평가**: 이 전략은 MKT-005를 대체하는 것이 아니라 **확장**합니다. MKT-005의 선별 Deep Analysis를 상위 레이어로 유지하면서, 그 아래에 **전 종목 자동 스캔 레이어**를 깔아 "세계 유일의 전 종목 크립토 등급 서비스"로 포지셔닝합니다.

### 3.2 Tier 1: Auto Scan — 전 종목 자동 등급 시스템

이것이 MKT-006의 핵심이자 MKT-005 대비 근본적 차이점입니다.

**BCE Grade 체계 (6단계):**

| Grade | 의미 | 기준 (자동 산출) | 예상 비율 |
|:---:|------|------|:---:|
| **A** | Excellent | 모든 데이터 확보 + 높은 투명성 + 강한 펀더멘탈 | ~5% |
| **B** | Good | 대부분 데이터 확보 + 양호한 투명성 | ~15% |
| **C** | Fair | 부분적 데이터 + 일부 투명성 이슈 | ~25% |
| **D** | Poor | 최소 데이터만 확보 + 낮은 투명성 | ~25% |
| **F** | Fail | 레드 플래그 다수 + 매우 낮은 투명성 | ~10% |
| **⚠️ UR** | **Unrated** | **"투명성 부족으로 등급 산출 불가"** | **~20%** |

**투명성 라벨 (Transparency Label) — 5단계:**

```
🟢 OPEN       팀 공개 + 코드 오픈소스 + 토큰 분배 공개 + 감사 완료
🔵 MOSTLY     팀 공개 + 코드 부분 공개 + 토큰 분배 공개
🟡 PARTIAL    팀 부분 공개 + 코드 비공개 + 기본 정보만 공개
🟠 LIMITED    팀 익명 + 최소 정보만 공개
🔴 OPAQUE     "분석에 필요한 정보를 확인할 수 없음"
```

**⚠️ UR (Unrated)와 🔴 OPAQUE가 CEO 지시의 핵심입니다.** 이 라벨이 붙은 프로젝트의 관계자들은 다음과 같은 반응을 보일 것입니다:

```
[프로젝트 관계자가 BCE Lab 페이지에서 자기 프로젝트를 발견]

"우리 프로젝트가 ⚠️ Unrated (투명성: 🔴 OPAQUE)라고?!"
    │
    ├── 반응 A (70%): "어떻게 하면 등급을 받을 수 있지?"
    │   → BCE Lab에 자발적 접촉
    │   → 데이터 제공 / 인터뷰 요청
    │   → Discovery Report 또는 Full Report 의뢰 가능성
    │
    ├── 반응 B (20%): 소셜에서 BCE Lab 언급 (긍정/부정)
    │   → 화제성 + 트래픽 유입
    │   → "크립토의 무디스가 우리를 평가했다" 효과
    │
    └── 반응 C (10%): 무시
        → 손실 없음
```

### 4.2 핵심 메시지 (화제성 최적화)

**1차 메시지 — 론칭 시 (바이럴 트리거):**

> "We just rated every single token on every centralized exchange. Some projects scored well. Some didn't. And some were too opaque to rate at all."

이 한 문장이 크립토 트위터에서 폭발적 반응을 일으킬 수 있는 이유:
- 모든 프로젝트의 관계자가 **자기 프로젝트 등급을 확인하러** 방문
- 높은 등급을 받은 프로젝트는 **자랑하며 공유**
- 낮은 등급/Unrated를 받은 프로젝트는 **반박/해명**하며 언급
- 투자자들은 **자기 포트폴리오 등급을 확인**하러 방문

어느 반응이든 **BCE Lab으로의 트래픽과 화제성**입니다.

**2차 메시지 — 프로젝트 관계자 대상:**

> "Your project is currently Unrated due to insufficient transparency. Want to change that? We're listening."

**3차 메시지 — 투자자 대상:**

> "Before you invest, check the BCE Grade. A-rated projects have 100% transparency. F-rated ones... don't."

## 5. "투명성 라벨"의 전략적 가치 — 프로젝트 관계자 유인 효과

### 5.1 왜 프로젝트 팀이 반응하는가

전통 금융에서 Moody's/S&P/Fitch 등급은 기업이 자본시장에 접근하기 위한 필수 요소입니다. 크립토에는 이런 표준화된 등급 시스템이 없습니다. BCE Lab이 이 공백을 채우면:

**프로젝트 입장에서의 가치:**

| 시나리오 | 프로젝트의 반응 | BCE Lab에 대한 가치 |
|----------|-------------|-------------------|
| A/B 등급 획득 | "BCE Lab rated, Grade A" 배지를 웹사이트/피치덱에 사용 | 브랜드 권위 + 무료 홍보 |
| C/D 등급 획득 | 등급 개선 방법 문의 → 데이터 자발 제공 | 데이터 수집 비용 $0 + B2B 리드 |
| F 등급 획득 | 반박/해명 → 소셜 화제 | 화제성 + 트래픽 |
| ⚠️ Unrated | "왜 등급이 없는가?" → 투명성 개선 후 재스캔 요청 | 프로젝트가 자발적으로 투명성 향상 |

**핵심 인사이트**: 등급이 낮거나 없는 프로젝트일수록 BCE Lab에 더 적극적으로 접촉합니다. 이것은 **프로젝트가 BCE Lab에게 데이터를 무료로 제공**하게 만드는 구조입니다. MKT-005에서 우려한 "데이터 수집 어려움"이 이 모델에서는 **프로젝트 측 자발적 데이터 제공**으로 해결됩니다.

### 5.2 "Claim Your Rating" 프로세스

프로젝트 관계자가 BCE Lab에 접촉 시 표준 프로세스:

```
[프로젝트 관계자 접촉]
        │
        ▼
Step 1: "Claim Your Project" 페이지
        · 프로젝트 관계자 본인 확인 (팀 이메일/공식 X 계정 DM)
        · 무료
        │
        ▼
Step 2: 기본 정보 제출 (Self-Report)
        · 팀 정보, 토큰 분배 상세, 로드맵, 감사 보고서
        · 무료 — 이 정보만으로 투명성 라벨 업그레이드 가능
        │
        ▼
Step 3: BCE Lab 검증
        · 제출 정보 vs 온체인 데이터 크로스체크
        · 검증 완료 시 등급 재산출 + 투명성 라벨 업그레이드
        · 무료 (Tier 1 Auto Scan 수준)
        │

**비용 구조**: Step 1-3은 완전 무료입니다. 프로젝트가 데이터를 제공하는 대가로 투명성 라벨을 업그레이드하는 **가치 교환**입니다. BCE Lab은 무료로 데이터를 획득하고, 프로젝트는 무료로 등급을 개선합니다. Step 4에서만 수익이 발생하며, 이것은 자연스러운 B2B 전환입니다.

## 6. 수익 모델 재설계: 화제성 → 수익화 경로

### 6.1 초기 전략 = 화제성 최우선, 수익은 후순위

CEO 지시대로 시장 진입 초기에는 유료 판매를 늘리는 것이 아니라 **화제성이 높은 서비스**가 되는 것이 목표입니다.

**Phase별 우선순위 전환:**

```
Phase 1-2 (Month 1-6):  화제성 90% / 수익 10%
Phase 3   (Month 7-12): 화제성 60% / 수익 40%
Phase 4+  (Month 13+):  화제성 40% / 수익 60%
```

### 6.3 무디스 모델 vs BCE Lab 모델 비교

| 차원 | Moody's/S&P (전통금융) | BCE Lab (크립토) |
|------|------|------|
| 등급 부여 | 기업이 의뢰 (Issuer-Pays) | **자동 스캔 후 Claim 유도** |
| 수수료 | 등급 의뢰비 $500K-$1M+ | **$299-$1,999** (크립토 규모에 맞춤) |
| 등급 공개 | 일부 유료 | **전체 무료 공개** (화제성 극대화) |
| 방법론 | 부분 공개 | **100% 공개** (신뢰 구축) |
| 이해충돌 | Issuer-Pays 비판 존재 | **Auto Scan이 기본이므로 이해충돌 최소화** |

**핵심 차이**: 전통 금융의 Issuer-Pays 모델은 이해충돌 비판을 받습니다. BCE Lab은 **자동 스캔으로 전 종목을 무료 평가한 후, 프로젝트가 원할 때만 유료 서비스를 제공**하므로 이해충돌 구조를 피합니다.

### 7.3 Auto Scan 등급 산출 공식

```
BCE Grade Score (0-100) = 자동 산출 가중 합계

1. 시장 건전성 (25%)
   · 시총 규모 점수 (로그 스케일 정규화)
   · 24h 거래량 / 시총 비율 (적정 범위: 5-30%)
   · 가격 변동성 (30일 표준편차 역수)

2. 투명성 (30%)
   · 팀 공개 여부: +6점
   · 코드 오픈소스: +6점
   · 토큰 분배 공개: +6점
   · 감사 완료: +6점
   · 컨트랙트 검증: +6점
   (30점 만점, 미확인 항목은 0점)

3. 홀더 분산도 (20%)
   · Top 10 홀더 집중도 역수 (낮을수록 좋음)
   · 총 홀더 수 (로그 스케일)

4. 개발 활동 (15%)
   · GitHub 커밋/주 (있으면 점수, 없으면 N/A → 가중치 재분배)
   · PR/이슈 활동

5. 거래소 상장 다양성 (10%)
   · 상장 CEX 수
   · Binance/Coinbase 상장 여부 가산

등급 기준:
A: 80-100 | B: 65-79 | C: 50-64 | D: 35-49 | F: 0-34
UR: 투명성 점수 < 10 (5개 항목 중 2개 미만 확인)
```

| **등급 조작 시도** (프로젝트가 가짜 데이터 제출) | MEDIUM | Step 3 온체인 크로스체크 필수. Claim 데이터 vs 실제 온체인 불일치 시 "Disputed" 라벨 |

| **Unrated 프로젝트의 부정적 PR** | MEDIUM | "Unrated ≠ 나쁜 프로젝트. 단지 투명성 데이터가 부족할 뿐" 메시지 일관 유지 |
| **이해충돌 비판** (B2B로 돈 받고 등급 올려주는 거 아닌가) | HIGH | Tier 1 Auto Scan은 절대 변경 불가 (돈을 내도). B2B는 추가 데이터에 의한 재산출일 뿐, 점수 보장 없음. 방법론 문서에 명시 |

**이해충돌 방어 원칙 (절대 규칙):**

```
1. Tier 1 Auto Scan 등급은 어떤 경우에도 수동 변경 불가
2. B2B Verified 서비스는 "추가 데이터 제출 → 재스캔"이지 "등급 보장"이 아님
3. B2B 고객이 데이터를 제출해도 Auto Scan 공식에 의해 낮은 등급이 나올 수 있음
4. 이 원칙을 방법론 문서와 약관에 명시

## 12. Conclusion

이 전략의 본질은 **관점의 전환**입니다.

MKT-004까지는 "우리의 보고서를 사줄 고객을 어떻게 찾을까"였습니다. MKT-006은 "전 세계 모든 크립토 프로젝트가 우리를 찾아오게 만들자"입니다.

2,000개 이상의 토큰에 등급을 부여하면, 2,000개의 프로젝트 팀이 잠재적 관심 대상이 됩니다. 그 중 20%인 400개 이상이 "투명성 부족으로 등급 불가"를 받으면, 이 프로젝트들은 등급을 받기 위해 자발적으로 BCE Lab에 데이터를 제공하고, 그 과정에서 B2B 수익이 자연스럽게 발생합니다.

동시에, 모든 투자자가 자기 포트폴리오의 등급을 확인하러 방문하고, 높은 등급의 프로젝트는 BCE Lab 등급을 자랑하며 공유합니다. 이 모든 활동이 BCE Lab의 화제성과 트래픽을 만들고, 이것이 뉴스레터 구독, 보고서 판매, 레퍼럴의 기반이 됩니다.

**"Every Token Traded. Every Token Rated."** — 이것이 BCE Lab을 크립토 리서치 시장의 표준으로 만드는 전략입니다.
