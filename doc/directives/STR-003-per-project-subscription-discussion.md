# STR-003: 종목별 구독 상품 모델 — C-Level 임원 논의록

> **주관**: agent-ceo (연구소장)
> **참석**: agent-cro, agent-coo, agent-cmo
> **일시**: 2026-04-09
> **상태**: Decision Required → Board 최종 승인 필요
> **Goal Ancestry**: Company Mission → STR-002 → 이 논의

---

## 1. 안건

Board(Human)가 제안한 **"종목별 구독"** 상품 모델의 도입 여부를 논의한다.

> *"이 보고서는 '종목별 구독' 상품으로 파는게 어떨까"* — Board

현재 상품 체계는 `single_report`(단건), `subscription`(전체 구독), `bundle`(묶음)의 3종이다. 여기에 **특정 크립토 프로젝트를 축으로 한 구독 상품**을 추가하는 방안을 검토한다.

---

## 2. CEO 의견 (agent-ceo)

**찬성, 핵심 상품으로 채택 권고**

종목별 구독은 우리 연구소의 차별화 포인트가 될 수 있다. 기존 크립토 리서치 서비스들이 "전체 구독" 모델에 집중하는 반면, 특정 프로젝트에 대한 심층 추적을 원하는 투자자 수요는 충분히 존재한다.

다만 다음 사항을 고려해야 한다:
- 프로젝트당 최소 3종 보고서(ECON + MAT + FOR)가 지속 발행되어야 상품 가치가 유지됨
- 프로젝트가 소멸하거나 비활성화될 경우의 구독자 보호 정책이 필요
- 가격 설정은 전체 구독 대비 충분히 매력적이되, 3개 이상 종목 구독 시 전체 구독으로 업그레이드 유인이 작동해야 함

---

## 3. CRO 의견 (agent-cro)

**찬성, 생산 역량 기반 단계적 도입 권고**

연구 관점에서 종목별 구독은 자연스러운 구조다. 이미 프로젝트 단위로 보고서를 생산하므로 상품 묶음의 축을 프로젝트로 설정하는 것은 논리적이다.

**우려 사항:**
- 현재 연구원 4명으로 주 3개 프로젝트를 커버하면, 월 12~15개 프로젝트가 누적됨
- 6개월 후 약 70~90개 프로젝트의 갱신 주기가 겹칠 수 있음
- 갱신 부담을 고려하여 동시 활성 프로젝트 상한선(예: 50개)을 설정해야 함

**제안:**
- Phase 1 (Q2): 10개 프로젝트로 시작, 종목별 구독 파일럿
- Phase 2 (Q3): 30개로 확대, 자동화 도구 구축
- Phase 3 (Q4): 50개 목표, 구독자 피드백 반영 최적화

---

## 4. COO 의견 (agent-coo)

**찬성, 기술 인프라 준비 후 도입 권고**

운영 관점에서 종목별 구독 도입에는 다음 인프라가 필요하다:

**필수 구축 사항:**
1. `tracked_projects` 테이블 — 프로젝트별 상태, 갱신 주기, 마지막 발행일 추적
2. `project_subscriptions` 테이블 — 사용자별 종목 구독 관리
3. 자동 스케줄러 — 갱신 주기 도래 알림 자동 발송 (CRO + 담당 연구원)
4. 구독자 알림 시스템 — 해당 종목 보고서 발행 시 구독자에게만 알림
5. 종목별 대시보드 페이지 — 프로젝트별 모든 보고서를 한 곳에서 열람

**운영 비용 추정:**
- DB/인프라: 추가 비용 미미 (Supabase 무료 티어 범위 내)
- 개발 공수: 약 2주 (데이터 엔지니어 전담)
- 유지보수: 기존 파이프라인에 종목 필터 추가 수준

---

## 5. CMO 의견 (agent-cmo)

**강력 찬성, 마케팅 차별화 최적**

마케팅 관점에서 종목별 구독은 매우 매력적인 상품이다:

**장점:**
- **낮은 진입 장벽**: 전체 구독($99/월) 대비 종목 구독($19~29/월)은 신규 사용자 유치에 유리
- **업셀링 경로 명확**: 1종목 → 3종목 → 전체 구독으로의 자연스러운 업그레이드
- **타겟 마케팅 가능**: "이더리움 투자자를 위한 전문 분석 구독" 등 세분화된 메시지
- **구전 효과**: "이 프로젝트 분석이 좋았다"는 구체적 추천이 가능
- **리드 마그넷**: 특정 종목의 첫 보고서를 무료로 제공하여 전환 유도

**제안 가격 체계:**

| 상품 | 가격 | 포함 내용 |
|------|------|----------|
| 종목별 구독 (1종목) | $19/월 | 해당 프로젝트의 ECON + MAT + FOR 전체 |
| 종목별 구독 (3종목) | $49/월 | 3개 프로젝트 선택 (14% 할인) |
| 종목별 구독 (5종목) | $69/월 | 5개 프로젝트 선택 (27% 할인) |
| 전체 구독 | $99/월 | 모든 프로젝트 + 매크로 리포트 |

3종목 이상 구독 시 전체 구독과의 가격 차이가 좁아져 업그레이드 유인이 자연 발생.

## 6. 종합 결론 및 권고안

### 6.1 만장일치 합의 사항

1. **종목별 구독 도입에 전원 찬성**
2. Board의 제안은 상품 전략적으로 타당하며, 기존 생산 파이프라인과 자연스럽게 정합
3. 단계적 도입이 리스크 관리에 적합

### 6.2 임원 합의 실행안

| 단계 | 기간 | 목표 | 핵심 액션 |
|------|------|------|----------|
| Phase 1 — 파일럿 | Q2 2026 (4~6월) | 10개 프로젝트, 종목별 구독 출시 | DB 스키마 구축, 종목 페이지 개발, 첫 3주 9개 프로젝트 발굴 |
| Phase 2 — 확장 | Q3 2026 (7~9월) | 30개 프로젝트, 자동화 강화 | 스케줄러 자동화, 구독자 알림, 업셀링 퍼널 구축 |
| Phase 3 — 최적화 | Q4 2026 (10~12월) | 50개 프로젝트, 수익 안정화 | 구독자 피드백 반영, 가격 최적화, 연구원 충원 검토 |

### 6.3 가격 체계 (합의안)

- 종목별 1종목: **$19/월** (BTC/ETH/USDT/USDC)
- 종목별 3종목: **$49/월**
- 종목별 5종목: **$69/월**
- 전체 구독: **$99/월** (기존)
- 연간 결제 시: 20% 할인

### 6.4 구독자 보호 정책

- 프로젝트가 `monitoring_only` 또는 `archived` 전환 시, 해당 구독자에게 다른 활성 프로젝트로 변경 가능
- 프로젝트 러그풀/스캠 확정 시, 해당 종목 구독료 잔여 기간 환불 또는 크레딧 전환
- 갱신 주기 내 보고서 미발행 시 자동 연장 (발행일까지 구독 기간 정지)
