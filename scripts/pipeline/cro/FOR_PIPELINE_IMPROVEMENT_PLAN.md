## 3. 격차 분석 결론

BCE FOR 보고서는 **포렌식 조사 깊이**와 **방법론적 엄격성**에서 참조 보고서보다 우수합니다. 특히 워시트레이딩 정량화, 정보비대칭 탐지, 리스크 매트릭스, 교차검증 프레임워크는 참조 보고서에 없는 고급 분석입니다.

그러나 **파생상품 실시간 데이터**, **기술적 분석 지표**, **거래소 미시구조 분석**, **실행 가능한 트레이딩 전략**에서 참조 보고서가 크게 앞서 있습니다.

핵심 격차 요약:
1. **파생상품 데이터 파이프라인 부재** — OI, 펀딩레이트, 청산 데이터 미수집
2. **기술적 지표 계산 엔진 부재** — RSI, SMA, EMA, 피보나치, 변동성 미산출
3. **거래소별 미시구조 데이터 부재** — 오더북, 거래소별 가격/볼륨 미분리
4. **실행 전략 프레임워크 부재** — 레버리지 권고, 진입/퇴출 시나리오 미제공

### Phase 1: 파생상품 데이터 수집 파이프라인 구축 (우선순위: 최고)

**목표**: FOR 보고서의 Chapter 5 (Derivatives & Supply-Side Pressure) 데이터 품질을 참조 보고서 수준으로 끌어올림

**4.1.1 CoinGlass API 통합**
- 상태: 이미 candidate로 등록됨 (data_source_registry)
- 수집 대상: Futures OI, 펀딩레이트, 청산 히트맵, 롱/숏 비율
- 엔드포인트:
  - `/futures/openInterest` — 거래소별 OI
  - `/futures/funding_rate` — 실시간 펀딩레이트
  - `/futures/liquidation` — 청산 데이터

- 작업: collector_coinglass.py 생성, validate → integrate
- 예상 소요: 2일

**4.1.2 청산 클러스터 계산 엔진**
- 가격대별 청산 집중 지점 산출 알고리즘 구현
- OI 분포 + 레버리지 배수 기반 3-tier 클러스터 맵핑
- 스퀴즈 확률 모델: 펀딩레이트 극단값 + OI 누적 + 볼륨 조건
- 신규 파일: `scripts/pipeline/analytics/liquidation_engine.py`
- 예상 소요: 3일

**4.1.3 gen_text_for.py Chapter 5 대폭 강화**
